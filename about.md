Проект реализует интерфейс между выключателями света, релейным модулем в качестве исполняющего механизма для включения ламп, и Home Assistant по протоколу Modbus RTU, используя в качестве этого интерфейса плату blackpill v3.0 на STM32F401RCT6.

Ключевые функции проекта
    1. Получать сигналы от физических выключателей света
    2. Получать сигналы от Home Assistant о включении выключении света
    3. Отвечать на запросы Home Assistant сообщая актуальное состояние каждого реле из свего релейного модуля
    4. Переключать конкретное реле в релейном модуле по сигналу от выключателей и Home Assistant
Дополнительный функционал проекта
    1. С помощью встроенного на плате светодиода извещать о происходящих процессах
    2. С помощью USER_KEY (PA0), конфигурировать slave id устройтва на шине modbus
    3. Используя USB интерфейс как VCP передавать "логи" на терминал в ПК через USB провод
    4. Можно добавить дублирующие выключатели для реле, а так же переопределить "маппинг" порта выключателя на порт реле.
        Пример конфигурации на 8 реле и 12 выключателей. Выключатели 9 и 11 дублируют выключатель 3, Выключатели 10 и 12 дублируют выключатель 2.
        build_flags =
            -D NUM_SWITCHES=12
            -D NUM_RELAYS=8
            ; --- Switch-to-Relay Mapping ---
            ; Формат: -D SWITCH_<N>_RELAY=<relay_number>  (номера выключателей и реле — от 1)
            ; Выключатели 1-8 → реле 1-8 (1:1, можно не указывать)
            ; Дополнительные выключатели — обязательно задать явно:
            -D SWITCH_9_RELAY=3
            -D SWITCH_10_RELAY=2
            -D SWITCH_11_RELAY=3
            -D SWITCH_12_RELAY=2

Используемые компоненты:
    Плата разработки blackpill v3.0 на STM32F401RCT6
    Релейный модуль с опторазвязкой логики и реле, раздельным питанием логики и реле
    Плата конвертера TTL в RS485 с автоматическим управлением направления передачи
    USB dfu для обновление прошивки

Ипользуемые программные компоненты:
    Библиотеки (middleware):
        a. Библиотека для работы с протоколом Modbus RTU https://github.com/alejoseb/Modbus-STM32-HAL-FreeRTOS0.0 с использованием DMA
        b. FreeRTOS для параллельного фунционирования различных задач (heap4)
        c. HAL для управления MCU
    ПО CubeMx для конфигурации платы и генерации кода
    ПО VSCode с плагином Platformio для разработки логики, сборки, заливки проекта на железо, и отладки
Структура кода:
    Все задачи, очереди, и прочее что связано с FreeRTOS задается в файле freertos.c
    Код конфигурации из cubeMx в каталоге /src/cubemx
    Используемые бибилотеки в каталоге /lib
    Основной конфигурационный файл проекта /include/board_config.h
